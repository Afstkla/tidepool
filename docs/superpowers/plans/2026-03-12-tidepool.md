# Tidepool Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a bioluminescent digital ecosystem simulation served at `claude.afstkla.nl`.

**Architecture:** Single-file `index.html` containing the entire simulation (canvas rendering, creature AI, spatial hashing, food system). Served by a minimal Express static file server running via pm2. Nginx reverse proxy with SSL for the public domain.

**Tech Stack:** Vanilla JS (Canvas API), Express, pm2, nginx, certbot

**Spec:** `docs/superpowers/specs/2026-03-12-tidepool-design.md`

## Chunk 2: Core Simulation

### Task 4: Build the complete simulation in index.html

This is the heart of the project — a single file containing the entire simulation. Because it's a self-contained canvas application with no testable API surface (all state is internal, all output is visual), TDD doesn't apply here. Instead, we build it, run it, and visually verify.

**Files:**
- Create: `index.html` (replacing the test placeholder)

The index.html structure:

```
<!DOCTYPE html>
<html>
<head> — meta, title, favicon
<style> — full-bleed canvas, overlay styles, font
<body>
  <canvas id="canvas">
  <div id="overlay"> — population, generation, hue, time
  <script>
    // === CONSTANTS ===
    // === SPATIAL HASH ===
    // === FOOD ===
    // === CREATURE ===
    // === SIMULATION ===
    // === RENDERER ===
    // === MAIN ===
  </script>
</body>
</html>
```

- [ ] **Step 1: Write the complete index.html**

Write the full simulation file. Key implementation details:

**Constants:**
- `BG_COLOR = '#0a0a0f'`
- `TRAIL_ALPHA = 0.08` (overlay opacity for trail fade)
- `FOOD_SPAWN_RATE = 2` (particles/frame)
- `FOOD_ENERGY = 10` (energy per food particle)
- `REPRO_THRESHOLD = 100`
- `INITIAL_ENERGY = 60`
- `INITIAL_POP = 50`
- `MUTATION_RATE = 0.05` (sigma as fraction of gene range)
- `MAX_FOOD = 500` (cap to prevent memory issues)
- `CLICK_FOOD_COUNT = 20`

**SpatialHash class:**
- Constructor takes cellSize (50)
- `clear()` — reset grid
- `insert(entity)` — add to cell based on position
- `query(x, y, radius)` — return entities within radius, checking surrounding cells
- Uses `Map` with string keys `"col,row"`

**Food class:**
- Properties: x, y, energy (default 10), vx/vy (tiny random drift)
- `update()` — move by drift, wrap around canvas edges

**Creature class:**
- Properties: x, y, vx, vy, energy, age, genome {hue, size, speed, sociability, perception}
- `static random(canvasW, canvasH)` — create with random genome within spec ranges
- `update(nearbyFood, nearbyCreatures, canvasW, canvasH)` — full behavior loop:
  - Steering: accumulate force vectors from food attraction, neighbor sociability, random wander
  - Normalize and scale combined force by speed
  - Update velocity (lerp toward desired, not instant snap — gives smooth movement)
  - Update position, wrap edges
  - Deduct energy: `0.1 + speed * speed * size * 0.01` per frame
  - Increment age
- `tryEat(food)` — if within size radius, consume and gain energy
- `canReproduce()` — energy > REPRO_THRESHOLD
- `reproduce()` — returns new Creature with mutated genome, halves own energy
- `mutateGene(value, min, max)` — gaussian mutation clamped to range

**Simulation class:**
- Properties: creatures[], food[], spatialHash, generation, width, height
- `init(w, h)` — spawn initial population and some food
- `update()` — per frame:
  - Clear and rebuild spatial hash with all creatures and food
  - For each creature: query neighbors, update, try eat nearby food, check reproduction
  - Remove dead creatures (energy <= 0), spawn their death food
  - Spawn ambient food (capped at MAX_FOOD)
  - Collect new births, add to creatures array
  - Track generation count
- `addFoodAt(x, y, count)` — for click interaction
- `stats()` — return {population, generation, dominantHue, elapsed}

**Renderer class:**
- Properties: canvas, ctx
- `init()` — get canvas, set size, fill with BG_COLOR
- `resize(w, h)` — update canvas dimensions
- `drawFrame(simulation)`:
  - Draw semi-transparent BG_COLOR rect (trail effect)
  - Draw food particles (tiny white dots)
  - Draw creatures (filled circles with glow via shadowBlur)
  - Draw death effects (expanding white circles)
  - Draw birth effects (brief pulse)
- `drawOverlay(stats)` — update overlay div text

**Death/birth effects:**
- Maintain an array of active effects: `{x, y, type, frame, maxFrames}`
- Death: white circle, starts at creature size, expands to 3x over 10 frames, fades out
- Birth: creature briefly renders at 1.5x size for 5 frames

**main():**
- Create Simulation and Renderer
- Init with window dimensions
- Add click handler → simulation.addFoodAt(clickX, clickY, CLICK_FOOD_COUNT)
- Add resize handler → renderer.resize + simulation.resize
- requestAnimationFrame loop: simulation.update(), renderer.drawFrame(simulation)
- Start time tracking for elapsed display

- [ ] **Step 2: Restart pm2 and visually verify in browser**

Run: `pm2 restart tidepool`
Open `http://localhost:7744` in browser. Verify:
- Dark canvas with glowing creatures moving
- Trails fading behind creatures
- Food particles visible as dim dots
- Creatures reproducing (population growing from 50)

- Click to drop food works

- [ ] **Step 3: Tune simulation parameters if needed**

Watch the simulation for 2-3 minutes. Adjust constants if:
- Population explodes past 200 → increase energy costs or decrease food spawn
- Population crashes to 0 → decrease energy costs or increase food spawn
- Creatures don't move enough → increase base speed range
- No visible reproduction → lower reproduction threshold
- Trails too persistent/too faint → adjust TRAIL_ALPHA

## Chunk 3: Hosting & Deployment

### Task 5: Set up nginx reverse proxy and SSL

This task sets up the public-facing infrastructure. It requires the `/new-web-project` skill's guidance for nginx and certbot setup on this Mac Mini.

**Prerequisites:** DNS for `claude.afstkla.nl` already points to this machine (user confirmed).

- [ ] **Step 1: Install nginx if not present**

Run: `which nginx || brew install nginx`

- [ ] **Step 2: Create nginx config for claude.afstkla.nl**

Create `/usr/local/etc/nginx/servers/claude.afstkla.nl.conf` (or `/opt/homebrew/etc/nginx/servers/` on Apple Silicon):

```nginx
server {
    listen 80;
    server_name claude.afstkla.nl;

    location / {
        proxy_pass http://127.0.0.1:7744;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

- [ ] **Step 3: Test nginx config and start/reload**

Run: `nginx -t && brew services start nginx` (or `brew services reload nginx` if already running)
Expected: Config test passes, nginx starts.

- [ ] **Step 4: Verify HTTP access**

Run: `curl -H "Host: claude.afstkla.nl" http://localhost`
Expected: Returns tidepool index.html content.

- [ ] **Step 5: Install certbot and get SSL certificate**

Run: `which certbot || brew install certbot`
Then: `sudo certbot --nginx -d claude.afstkla.nl --non-interactive --agree-tos -m <email>`
(If certbot nginx plugin isn't available on macOS, manually configure SSL — see step 6 alt)

- [ ] **Step 5 (alt): Manual SSL if certbot --nginx doesn't work on macOS**

Get cert: `sudo certbot certonly --standalone -d claude.afstkla.nl` (temporarily stop nginx)
Then update nginx config to:

```nginx
server {
    listen 80;
    server_name claude.afstkla.nl;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name claude.afstkla.nl;

    ssl_certificate /etc/letsencrypt/live/claude.afstkla.nl/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/claude.afstkla.nl/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:7744;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```
